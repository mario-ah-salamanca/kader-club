# ADR 0003: Prepare for Fantasy XI Without Building It Now

## Status

Accepted

## Context

Fantasy XI is a strong future differentiator because it lets users build a squad from cards they own.

However, the MVP must first validate collection entry, catalog trust, valuation, wishlist, and trade discovery.

## Decision

Do not build Fantasy XI in the MVP.

Prepare the data model and visual language so Fantasy XI is easy to add later.

## Consequences

### Positive

- MVP remains focused.
- The emotional collector layer is preserved in the roadmap.
- Data model avoids future rework.

### Negative

- Fun differentiation is delayed.
- Early product may feel more utility-driven.
- Public showcase should be considered as a lightweight bridge.
