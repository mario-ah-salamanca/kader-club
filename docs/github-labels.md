# GitHub Labels

Use these labels for issues and PRs.

## Type

| Label | Purpose |
|---|---|
| `type: feature` | User-facing feature |
| `type: bug` | Defect |
| `type: task` | Implementation or operational task |
| `type: docs` | Documentation work |
| `type: research` | Research or discovery |
| `type: decision` | Decision needed |
| `type: risk` | Risk tracking |

## Priority

| Label | Purpose |
|---|---|
| `priority: P0` | Must happen now |
| `priority: P1` | Important but not blocking immediate demo |
| `priority: P2` | Later or post-MVP |

## Area

| Label | Purpose |
|---|---|
| `area: product` | Product management |
| `area: design` | UX/UI |
| `area: engineering` | App implementation |
| `area: data` | Catalog/data quality |
| `area: pricing` | Valuation/pricing logic |
| `area: trade` | Wishlist/trade matching |
| `area: docs` | Documentation |
| `area: infrastructure` | CI/CD, deployment, environments |

## Status

| Label | Purpose |
|---|---|
| `status: blocked` | Cannot continue |
| `status: needs-review` | Needs review |
| `status: needs-decision` | Needs a decision |
