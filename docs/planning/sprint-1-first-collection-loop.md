# Sprint 1: First Collection Loop

## Sprint Goal

Build the foundation that lets a user create an account, search a small curated catalog, and add cards to their collection.

## Sprint Duration

1–2 weeks depending on capacity.

## Sprint Tasks

1. Create Supabase schema for profiles, players, clubs, leagues, card sets, card variants, and user cards.
2. Implement Supabase Auth.
3. Implement protected app layout.
4. Implement profile onboarding.
5. Build basic navigation.
6. Seed 100–300 football cards manually or through CSV.
7. Build card search page.
8. Build card detail page.
9. Build add-to-collection form/modal.
10. Build collection list/grid page.
11. Add RLS policies.
12. Add integration tests for protected access.
13. Add empty-state and success-state UX.
14. Add basic tracking events.

## Sprint Demo Script

1. Create a new account.
2. Complete profile with display name, country, favorite club, and trade availability.
3. Search for a football card.
4. Open card detail.
5. Add card to collection with condition, quantity, purchase price, and trade status.
6. See card in My Collection.
7. Confirm another user cannot edit it.

## Definition of Done

- [ ] Demo script can be completed.
- [ ] Auth works.
- [ ] RLS prevents cross-user access.
- [ ] Seed catalog contains at least 100 usable cards.
- [ ] User can add and view cards.
- [ ] Docs are updated.
