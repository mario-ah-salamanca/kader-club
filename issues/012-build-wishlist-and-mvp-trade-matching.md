# Build wishlist and MVP trade matching

## Goal

Turn collection data into discovery value.

## Acceptance Criteria

- [ ] User can create wishlist entries.
- [ ] User can mark owned cards with trade status.
- [ ] System can generate one-way matches.
- [ ] System can generate two-way matches.
- [ ] No payment, escrow, checkout, or shipping logic exists.
