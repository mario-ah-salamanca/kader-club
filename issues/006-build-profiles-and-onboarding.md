# Build profiles and onboarding

## Goal

Collect basic profile data that supports trust and future trade matching.

## Fields

- Display name
- Country
- Favorite club
- Bio
- Trade availability

## Acceptance Criteria

- [ ] User can create/update profile.
- [ ] Profile is linked to authenticated user.
- [ ] User cannot edit another user's profile.
- [ ] Empty and success states exist.

## Labels

`type: feature`, `priority: P0`, `area: engineering`, `area: design`
